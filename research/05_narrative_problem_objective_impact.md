## TL;DR — Elevator Pitch (1 ย่อหน้า)

พริก (*Capsicum annuum*) เป็นพืชเศรษฐกิจสำคัญที่ถูกคุกคามหนักจากโรค (แอนแทรคโนสอุบัติการณ์สูงถึง ~63% ในเอเชีย, ไวรัสใบหงิก PepYLCThV ในไทย) ขณะที่เกษตรกรเข้าถึงต้นพันธุ์ปลอดโรคได้น้อยมาก (ใช้เมล็ดรับรองเพียง ~7%) — การเพาะเลี้ยงเนื้อเยื่อ (tissue culture) คือทางออกที่ผลิตต้นพันธุ์ปลอดโรคได้ แต่พริกเป็นพืช **recalcitrant** (เพาะยาก, ตอบสนองต่ำ, แปรปรวนสูงข้าม genotype) และการ "ตรวจติดตามสภาพ culture" ยังพึ่งสายตามนุษย์ซึ่ง subjective, ใช้แรงงานเข้มข้น, scale ไม่ได้, และความแม่นยำของสายตามนุษย์ใกล้ถึงเพดานแล้ว. **VitroVision** เติมช่องว่างนี้ด้วยระบบ computer vision ที่ถ่ายภาพผ่านขวดปิด (รักษา aseptic) เพื่อ (1) จำแนกสถานะพืช healthy/contaminated/dead และ (2) วัด phenotype เชิงปริมาณ 7 ค่า แล้วใช้ค่าเหล่านั้นเปรียบเทียบสูตร MS 5 สูตรแบบ objective — เป็นงาน CSBI ที่มีทั้ง novelty เชิงคอมพิวเตอร์ (ยังไม่มีใครทำ image-based phenotyping เฉพาะ Capsicum in vitro) และคำถามทางชีววิทยาที่ทดสอบได้ (BAP สูงเพิ่มจำนวนยอดแต่กดการยืดยอด/เพิ่ม disorder จริงไหม, สูตรไหนให้ vigor สูงสุด).

## 1. Problem — ปัญหา (เชิงเกษตร + เชิงเทคนิค)

เรื่องเล่านี้มี **2 ชั้นปัญหาซ้อนกัน** — ปัญหาชีววิทยา/เกษตร (ทำไมต้องเพาะเลี้ยงเนื้อเยื่อพริก) และปัญหาเทคนิค (ทำไม monitoring แบบเดิมไม่พอ)

### 1.1 ชั้นเกษตร — พริกสำคัญแต่เปราะบาง และต้นพันธุ์สะอาดหายาก

- **พริกเป็นพืชเศรษฐกิจระดับโลก** ปลูกกว่า ~3.69 ล้านเฮกตาร์ทั่วโลก, มี capsaicin เป็นสารออกฤทธิ์มูลค่าสูง (อาหาร/เครื่องสำอาง/เภสัช); *C. annuum* คือชนิดสำคัญสุดเชิงพาณิชย์ [ดู 03 §1]. สำหรับไทย พริกผูกกับวัฒนธรรมอาหารและรายได้เกษตรกร โดยเฉพาะภาคอีสาน [ดู 03 §6, §7].
- **โรคพืชคือศัตรูตัวฉกาจ** — แอนแทรคโนส (*Colletotrichum*) อุบัติการณ์เฉลี่ย ~63% ความรุนแรง ~68% ในแหล่งปลูกเอเชีย [ดู 03 §2]; ไวรัสใบหงิกเหลือง (ChiLCVD / PepYLCThV) ทำลายล้างรุนแรงในไทยและกำลังแพร่สู่ลาว/เวียดนาม [ดู 03 §2].
- **ช่องว่าง "ต้นพันธุ์สะอาด"** — มีเกษตรกรเพียง ~7% ที่ใช้เมล็ดพันธุ์รับรอง [ดู 03 §2] สะท้อนว่าระบบการผลิตขาดต้นพันธุ์ปลอดโรคอย่างเป็นรูปธรรม.
- **Tissue culture คือทางออกที่ยอมรับแล้ว** — ขยายพันธุ์โคลนจำนวนมาก, ผลิตต้นปลอดโรคจาก meristem (มีรายงานพริกปลอดไวรัส 80% ผ่าน meristem tip culture) [ดู 03 §3].

### 1.2 ชั้นชีววิทยา — พริกเป็นพืช recalcitrant (ทำให้ "การคัดแยกจำนวนมาก" จำเป็น)

- **Recalcitrance เป็นปัญหาเชิงระบบ** — งานทบทวน 50 ปีตอกย้ำว่า in vitro response ของพริกต่ำ, ไม่ reproducible ข้าม genotype, มักได้ shoot ผิดรูป (SAM deformation), protocol ที่มี "inefficient และ little reproducible" [ดู 01 §2, ref 7]. นี่คือ "เหตุผลหลัก" (Intro hook) ว่าทำไม Capsicum TC ต้องการการ monitor จำนวนมากและเป็นระบบ.
- **ความแปรปรวนสูงมาก** — *C. annuum* ได้ ~1.44 shoot/explant ส่วน *C. chinense* เกือบ 0 ในเงื่อนไขเดียวกัน [ดู 01 §2]. ต้องการ throughput สูงเพื่อจับ variance.
- **Disorder ที่ต้องคัดทิ้ง** — browning/phenolic, hyperhydricity (ใบ glassy/translucent), contamination ที่ทำ media เปลี่ยนสี — ทั้งหมดเป็น "จุดตาย" ที่ต้องตรวจให้ทันและถี่ [ดู 01 §2, §3].

### 1.3 ชั้นเทคนิค — การ monitor ด้วยตาคนคือคอขวด

การประเมินสภาพ culture ส่วนใหญ่ยังพึ่ง **สายตามนุษย์ + การทำลายตัวอย่าง (destructive endpoint)** ซึ่ง:
1. **Subjective** — ขึ้นกับผู้ประเมิน [ดู 03 §4]
2. **ใช้แรงงานเข้มข้น** และเป็นคอขวดหลักของ phenotyping/breeding [ดู 03 §4]
3. **Scale ไม่ได้** เมื่อจำนวนขวดมาก
4. **ใกล้เพดานความแม่นยำของสายตามนุษย์** — งาน plant disease severity ระบุว่าความแม่นยำของ visual estimate "เกือบถึงเพดาน" เพิ่มต่อยาก [ดู 03 §4, ref 20]
5. การเปิดขวดเพื่อดู/วัด **เสี่ยง contamination**

> **ปมของเรื่อง:** พริกเพาะยาก (recalcitrant) → ต้องคัดแยกจำนวนมาก → แต่เครื่องมือคัดแยก (สายตาคน) subjective/ช้า/ไม่ scale → **ต้องการระบบ objective, non-destructive, scalable** → นี่คือที่ที่ VitroVision ยืน [ดู 01 §4 Gap, 03 §5].

## 2. Objectives — วัตถุประสงค์ (วัดผลได้)

### Primary Objective (หัวใจ — ต้องสำเร็จ)
**O1. พัฒนาระบบ computer vision ที่จำแนกสถานะพืช (healthy / contaminated / dead) จากภาพถ่ายผ่านขวดปิด** ให้แม่นยำในระดับที่ใช้แทนการประเมินด้วยตาคนได้ (วัดด้วย weighted F1 และ Cohen's κ เทียบ ground truth ที่ label โดยผู้เชี่ยวชาญ).

**O2. พัฒนา phenotyper วัด phenotype เชิงปริมาณ 7 ค่า** (`green_coverage_pct`, `leaf_color_index`, `shoot_count_cv`, `media_color_cv`, `texture_entropy`, `brown_coverage_pct`, `vigor_score`) แบบ non-destructive และตรวจสอบว่าแต่ละค่าผูกกับ response variable ทางชีววิทยาที่มี literature รองรับ (เช่น greener tissue = regeneration ดีกว่า [ดู 01 §3]).

### Secondary Objective (ต่อยอด — เพิ่มน้ำหนัก)
**O3. ใช้ phenotype ที่วัดได้เปรียบเทียบประสิทธิภาพสูตร MS 5 สูตร (A–E) แบบ objective** ด้วยสถิติ non-parametric (Kruskal-Wallis omnibus → Dunn/Mann-Whitney post-hoc + correction) เพื่อตอบคำถามชีววิทยาเรื่องผลของ BAP/NAA/IBA [ดู 02 §2].

**O4. สร้าง pipeline ที่ reproducible** — pin dependency, รวมโค้ด, seed คงที่, model registry ที่ผูก metric กับ commit/seed/dataset hash เพื่อให้กรรมการ/คนอื่น reproduce ผลได้ [ดู 04 §1, §2].

**O5. (ถ้าเป็นไปได้) ปิด active-learning loop** — ใช้ MC-Dropout uncertainty sampling คัดภาพที่โมเดลไม่มั่นใจมา label เพิ่ม เพื่อยกความแม่นด้วยภาพจำนวนน้อย [README §Features].

> หมายเหตุการจัดลำดับ: O1+O2 เป็น CS core (deliverable หลัก), O3 เป็น Biology payoff (ทำให้เป็น CSBI ไม่ใช่ CS เปล่า), O4 เป็นเงื่อนไขความน่าเชื่อถือสำหรับเวทีแข่ง, O5 เป็น nice-to-have.

## 3. Hypotheses — สมมติฐาน (ทดสอบได้ทั้งฝั่ง CS และ Biology)

> เขียนเป็น H (alternative) คู่กับ H₀ (null) ทุกข้อ เพื่อให้ทดสอบทางสถิติได้

### ฝั่ง CS — โมเดล/การวัดเชื่อถือได้แค่ไหน

**H1 (ความแม่นการจำแนก).** ระบบ CV จำแนก healthy/contaminated/dead ได้สอดคล้องกับ ground truth ของผู้เชี่ยวชาญในระดับ "substantial agreement" (Cohen's κ ≥ 0.70 บนภาพจริง).
- H1₀: κ ของระบบ ≤ ระดับ chance/บังเอิญ (หรือ ≤ เกณฑ์ที่ยอมรับ เช่น 0.40).
- *สถานะปัจจุบัน:* κ = 0.627 บนภาพ **สังเคราะห์** 181 ภาพ → ต้องทดสอบใหม่บนภาพจริง [README].

**H2 (phenotype สัมพันธ์กับ ground truth).** ค่า phenotype จาก CV (โดยเฉพาะ `shoot_count_cv`, `green_coverage_pct`, `vigor_score`) สัมพันธ์เชิงบวกกับการนับ/ให้คะแนนด้วยมือของผู้เชี่ยวชาญในระดับที่มีนัยสำคัญ (เช่น Pearson/Spearman, หรือ R² สำหรับ green→chlorophyll/vigor).
- H2₀: ไม่มีความสัมพันธ์ (r = 0) ระหว่าง CV feature กับ manual measurement.
- *ฐานความเป็นไปได้:* RGB→chlorophyll/health validated แล้วที่ R² ~0.67–0.90 ในพืชอื่น [ดู 01 §4, ref 11/13/15] — รองรับว่า `green_coverage_pct`/`leaf_color_index` เป็น proxy ที่มีฐาน ไม่ใช่ feature ลอยๆ.

### ฝั่ง Biology — สูตร MS ต่างกันให้ผลต่างกันอย่างไร

**H3 (BAP เพิ่มจำนวนยอด).** สูตร BAP สูง (C: 5 mg/L) ให้จำนวนยอด/proxy การแตกยอด (`shoot_count_cv`) มากกว่าสูตร BAP ต่ำ (B: 1 mg/L) และ control (A).
- H3₀: จำนวนยอดไม่ต่างกันระหว่าง A, B, C.
- *ฐาน:* BAP ~3–8 mg/L เป็นช่วง optimal ของ multiplication ในพริก, 5 mg/L ใช้บ่อย [ดู 02 §1.1].

**H4 (BAP สูงกดการยืดยอด — trade-off).** ที่ BAP 5 (C/D) ยอด "เพิ่มจำนวนแต่ไม่ยืด" → ความยาวยอด/ดัชนีการยืดต่ำกว่าที่ BAP ต่ำ แม้จำนวนยอดมากกว่า (diminishing return เชิงคุณภาพ).
- H4₀: ความยาวยอดไม่ต่างกันตามระดับ BAP.
- *ฐาน:* ที่ BAP 5 mg/L ใน cv. mathania ยอดเพิ่มจำนวนแต่ไม่ elongate [ดู 02 §1.2, ref 3].

**H5 (BAP สูงเพิ่ม disorder).** BAP 5 (C/D) แสดง hyperhydricity/necrosis/browning (จับด้วย `texture_entropy`, `brown_coverage_pct`) สูงกว่า BAP ต่ำ/control.
- H5₀: ระดับ disorder ไม่ต่างกันตาม BAP.
- *ฐาน:* BAP สูงเหนี่ยวนำ underdeveloped leaf, shoot-tip necrosis, hyperhydricity [ดู 02 §1.2, ref 13/14].

**H6 (ผลของ NAA บน BAP สูง — ratio).** การเติม NAA 0.05 (D เทียบ C ที่คุม BAP=5) เปลี่ยน phenotype (เช่น callus ที่โคน/ลด apical dominance) อย่างมีนัยสำคัญ.
- H6₀: D ไม่ต่างจาก C (NAA 0.05 เบาเกินจะเห็นผล).
- ⚠️ *ความเสี่ยง:* NAA 0.05 อาจต่ำเกินจะเห็น signal บน BAP 5 — มี literature เตือนไว้ [ดู 02 §2.2 ข้อ 3]. H6 มีโอกาส "ยอมรับ H₀" จริงโดยไม่ใช่ความผิดของระบบ.

**H7 (vigor ระหว่างสูตร).** `vigor_score` (composite 0–10) แตกต่างกันอย่างมีนัยสำคัญระหว่างสูตร และมีสูตรที่ให้ vigor สูงสุดอย่างชัดเจน (น่าจะเป็น B หรือ A สำหรับ "ต้นสมบูรณ์", C/D สำหรับ "จำนวนยอด" — ขึ้นกับนิยาม vigor).
- H7₀: vigor_score ไม่ต่างกันระหว่าง 5 สูตร (Kruskal-Wallis ไม่ significant).

> ⚠️ **คำเตือนการออกแบบ (สำคัญต่อ H ฝั่ง Biology):** สูตร E (IBA เดี่ยว) อยู่คนละ developmental stage (rooting) กับ A–D (multiplication) — **อย่าเทียบ E กับ C ด้วย endpoint เดียว (shoot count) ใน Kruskal-Wallis เดียวกัน** เพราะ E จะ "แพ้" โดยอัตโนมัติเพราะไม่ได้ออกแบบมาให้แตกยอด ไม่ใช่เพราะด้อยกว่า [ดู 02 §2.2 ข้อ 1]. แนะนำแยกเป็น "multiplication arm (A–D)" และ "rooting arm (E, อาจรวม D)".

## 4. Impact — ผลกระทบถึงใครบ้าง (เรียงตาม stakeholder)

> ตัวเลขประมาณการทุกตัวในส่วนนี้เป็น **การประเมินเชิงเหตุผล ไม่ใช่ผลที่วัดแล้ว** — ดูกล่อง ⚠️ ท้ายเอกสาร

1. **เกษตรกร / ผู้ผลิตต้นพันธุ์** (ผู้รับประโยชน์ปลายทาง) — ระบบช่วยให้การผลิตต้นพันธุ์พริกปลอดโรคในระดับ micropropagation **คัดกรองขวดเสีย (contaminated/dead) ได้เร็วและสม่ำเสมอ** ลดการสูญเสียวัสดุและเวลา. ประโยชน์เชื่อมโยงกับช่องว่าง "ต้นพันธุ์สะอาดเข้าถึงยาก" [ดู 03 §2].

2. **ห้องแล็บ TC / อุตสาหกรรม micropropagation** — ลด labor cost ของการตรวจติดตามรายขวด, เพิ่ม throughput, ได้ข้อมูล phenotype ต่อเนื่อง (ไม่ใช่จุดเดียวแบบ destructive). ตำแหน่งเทียบกับ "Phenomenon" [ดู 01 §4, ref 14] แต่ **low-cost, image-only, ไม่ต้อง scanner/hyperspectral** → เข้าถึงได้สำหรับแล็บเล็ก/ห้องเรียน.

3. **นักวิจัยพืช / TC** — phenotype เชิงปริมาณ objective ช่วยเปรียบเทียบสูตรอาหารและตอบคำถาม recalcitrance ได้แม่นกว่าการให้คะแนนด้วยตา; ขยายไปพืช recalcitrant อื่นได้.

4. **การศึกษา / open method** — เป็น open, reproducible pipeline ที่นักเรียน/ห้องเรียนทำซ้ำได้ (เป็นจุดขายของ YSC/ISEF: democratize plant phenotyping). [ดู 04 §4 quickstart]

5. **ระดับประเทศ (ความมั่นคงอาหาร/เศรษฐกิจพริกไทย)** — สนับสนุน supply ของต้นพันธุ์พริกปลอดโรค ซึ่งเชื่อมกับการลดความเสียหายจากโรค (แอนแทรคโนส/ไวรัส) ที่กระทบผลผลิตและรายได้ [ดู 03 §2].

### ขนาดของประโยชน์ (ประเมิน — ต้อง validate)
- **เชิงคุณภาพ (มั่นใจกว่า):** เปลี่ยน monitoring จาก subjective→objective, จาก destructive→non-destructive, จาก manual→scalable — ข้อนี้มีฐาน literature ชัด [ดู 03 §4–5].
- **เชิงปริมาณ (ประมาณการ — ⚠️):** *ถ้า* ระบบทำงานบนภาพจริงได้ตามเป้า อาจลดเวลา manual inspection ต่อรอบได้ระดับ "หลายเท่า" และลด human error ในการคัดขวดเสีย — แต่ **ยังไม่มีตัวเลข throughput/cost จริงในงานนี้** ห้ามเคลมเป็นเปอร์เซ็นต์เฉพาะเจาะจงจนกว่าจะวัด.

## 5. Expected Results — ผลที่คาดว่าจะได้รับ

### 5.1 Deliverables (สิ่งที่ส่งมอบ)
- **โมเดลจำแนก** EfficientNet-B0 (transfer learning) + checkpoint + metrics.json (ผูก git sha/seed/dataset hash) [ดู 04 §2.4].
- **Phenotyper 7 ค่า** + นิยาม + phenotype validation map (feature ↔ literature response) [ดู 01 §3].
- **ชุดข้อมูลภาพพริก in vitro ที่ label แล้ว** (เป้า 500+ ภาพจริง ภายใน ต.ค. 2569) + dataset manifest [README, 04 §1].
- **ระบบ/ dashboard** เปรียบเทียบสูตรพร้อมสถิติ (Kruskal-Wallis + post-hoc).
- **ข้อค้นพบทางชีววิทยา** เรื่องผล BAP/NAA/IBA ต่อ phenotype พริก (ตอบ H3–H7).
- **Reproducible pipeline** (environment lock, CLI/Makefile, model registry) [ดู 04 §2,§4].

### 5.2 ตัวเลขเป้าหมายที่ defensible (อิงผลปัจจุบัน)

| Metric | ตอนนี้ (ภาพสังเคราะห์) | เป้าภาพจริง (ประเมิน) | เหตุผล |
|---|---|---|---|
| Weighted F1 (classification) | 0.750 | **0.78–0.85** ถ้าได้ภาพจริง ≥500 + คุมแสง/มุม | ภาพจริงมี cue มากกว่า synthetic แต่ก็มี noise จริง |
| Cohen's κ vs ผู้เชี่ยวชาญ | 0.627 | **≥ 0.70** (เป้าที่ตั้งใน README) | เป็นเกณฑ์ substantial agreement; defensible ถ้า class balance ดี |
| R² (green/vigor proxy ↔ manual) | — | **0.65–0.85** | อิง RGB→chlorophyll ในพืชอื่น 0.67–0.90 [ดู 01 §4] |
| Human baseline κ (ครูที่ปรึกษา) | — | วัดเป็น reference | ใช้เทียบว่าระบบใกล้/เกินคนไหม [README objective] |

> **ข้อควรระวังเรื่องตัวเลข:** F1 0.75 / κ 0.627 ปัจจุบัน **มาจากภาพสังเคราะห์** — เป็น proof-of-pipeline ไม่ใช่ผลจริง. การตั้งเป้าภาพจริงควรเขียนเป็น "เป้า" พร้อมระบุว่ายังไม่ได้พิสูจน์ จนกว่าจะมีภาพจริง [README]. **อย่ารายงาน 0.627 ราวกับเป็นผลบนข้อมูลจริง** — กรรมการ ISEF จะถามทันที.

### 5.3 ผลชีววิทยาที่คาดว่าจะเห็น (สมมติฐานนำ — ต้องยืนยันด้วยการทดลองจริง)
- B/C ให้จำนวนยอด > A (H3 น่าจะ "ยอมรับ H").
- C/D ยอดสั้นกว่าแต่จำนวนมากกว่า (H4 trade-off — จุดขายของ CV ที่จับ trade-off นี้ได้).
- C/D มี disorder สูงกว่า (H5).
- D อาจไม่ต่างจาก C (H6 เสี่ยงยอมรับ H₀ เพราะ NAA เบา).

## 6. Reality Check — ความเป็นไปได้จริง (อ่านส่วนนี้ก่อน pitch)

### 6.1 ข้อจำกัด/ความเสี่ยงที่อาจทำให้ "เรื่องเล่าไม่จริง"

| # | ความเสี่ยง | ทำไมสำคัญ | สถานะ |
|---|---|---|---|
| RC1 | **ยังไม่มีภาพจริง** — metric ทั้งหมดมาจากภาพสังเคราะห์ 181 ภาพ | ถ้าภาพจริงไม่มา หรือมาน้อย เรื่องเล่าทั้งหมดเป็น "pipeline ที่ยังไม่พิสูจน์" | 🔴 critical — เป้า 500+ ภาพ ต.ค. 2569 [README] |
| RC2 | **n จำกัด (20/สูตร, 100 ขวด)** + loss จาก contamination/ตาย จะลด effective n | KW power ลด, post-hoc อ่อน; variance TC สูงอยู่แล้ว | 🟠 วางแผนเผื่อ loss ~20% [ดู 02 §2.3] |
| RC3 | **ขวดปิด ถ่ายยาก** — แสงสะท้อนผิวขวด, condensation, มุมซ้อน, ระบุยอดแยกกันยาก | กระทบ `shoot_count_cv`, segmentation, ความแม่น | 🟠 ต้องคุมแสง/มุม/มี ArUco ช่วย |
| RC4 | **dose-response BAP มีแค่ 2 จุด (1, 5)** — มองไม่เห็นรูปโค้ง/optimum | H4 (diminishing return) อ้างได้แค่ "ขึ้น/ลง" ไม่เห็นจุดพลิก | 🟠 พิจารณาเพิ่ม BAP กลาง 2–3 mg/L [ดู 02 §2.2 ข้อ 2] |
| RC5 | **E คนละ stage กับ A–D** — confound การวิเคราะห์ | ถ้าเทียบผิด ผลชีววิทยาตีความผิด | 🟠 แยก arm การวิเคราะห์ [ดู 02 §2.2 ข้อ 1] |
| RC6 | **D (NAA 0.05) อาจไม่เห็นผล** | H6 เสี่ยงเป็น null โดยไม่ใช่ความผิดระบบ | 🟡 พิจารณายก NAA เป็น 0.1–0.5 [ดู 02 §2.2 ข้อ 3] |
| RC7 | **Generalization** — โมเดล train จาก setup เดียว/ผู้ทำคนเดียว | อาจไม่ทรานสเฟอร์ไปแล็บ/กล้อง/พันธุ์อื่น | 🟡 ระบุเป็น limitation; ทดสอบ cross-condition ถ้าทำได้ |
| RC8 | **Ground truth / human baseline ยังไม่มี** | H1/H2 ทดสอบไม่ได้จนกว่าจะมี expert label | 🟠 ครูที่ปรึกษา label 28 ภาพ เป็น baseline [README] |
| RC9 | **phenotype เป็น proxy ไม่ใช่การวัดตรง** (เช่น green% ≠ chlorophyll จริง) | ต้อง validate correlation ไม่ใช่ assume | 🟡 มีฐาน R² ในพืชอื่น แต่ต้องวัดในพริกเอง [ดู 01 §3] |
| RC10 | **ตัวเลขเศรษฐกิจ macro ยังไม่ครบ** (มูลค่าตลาด, ผลผลิตไทย) | กระทบความหนักแน่นของ "ความสำคัญ" ใน Intro | 🟡 ต้องดึงจาก FAOSTAT/OAE [ดู 03 §7] |

### 6.2 จุดที่เรื่องเล่า "แข็งแรงจริง" (ใช้ได้เลย)
- ปัญหา recalcitrance + ช่องว่าง manual monitoring มี literature หนาแน่นรองรับ [ดู 01, 03].
- Novelty ชัด: ยังไม่มี image-based CV phenotyping เฉพาะ Capsicum in vitro + เชื่อม recalcitrance [ดู 01 §4].
- phenotype features มีฐานชีววิทยา (greener=better, RGB→chlorophyll validated) [ดู 01 §3,§4].
- การออกแบบสูตร A–D มี dose/ratio contrast ที่ตีความได้ [ดู 02 §2.1].

## ⚠️ ต้อง validate (เจ้าของงานตรวจก่อนใช้)

1. **ภาพจริง (RC1)** — ทั้งเรื่องเล่าตั้งอยู่บนสมมติว่าจะได้ภาพจริง ≥500 ภายใน ต.ค. 2569. ถ้ายังไม่ได้ ให้กรอบทุก metric เป็น "ผลบนภาพสังเคราะห์ (proof-of-pipeline)" อย่างชัดเจน **ห้ามนำเสนอ κ=0.627 ราวกับเป็นผลภาพจริง**.
2. **เป้า metric ภาพจริง (5.2)** — F1 0.78–0.85, κ≥0.70, R² 0.65–0.85 เป็น **ประมาณการของผม** อิงพืชอื่น/ผล synthetic ปัจจุบัน — ตรวจว่าตั้งเป้านี้ defensible กับ class balance และจำนวนภาพจริงที่จะได้.
3. **ผลชีววิทยาที่คาด (5.3 / H3–H7)** — เป็นทิศทางจาก literature (cv./species อื่นเป็นส่วนใหญ่) **ยังไม่ใช่ผลการทดลองของคุณ** — อย่าเขียนเป็นข้อสรุปจนกว่าจะรันจริง. โดยเฉพาะ H6 (D vs C) มีโอกาสเป็น null.
4. **การออกแบบสถิติ (RC2, RC5)** — ยืนยันว่าจะแยก multiplication arm vs rooting arm และวางแผน effective n เผื่อ loss; รายงาน effect size ไม่ใช่แค่ p-value [ดู 02 §2.4].
5. **dose-response & NAA (RC4, RC6)** — ตัดสินใจว่าจะเพิ่มสูตร BAP กลาง / ยก NAA ไหม **ก่อนเริ่มทดลอง** (เปลี่ยนทีหลังไม่ได้).
6. **ตัวเลขเศรษฐกิจ (RC10)** — ดึงมูลค่าตลาด/ผลผลิตจาก FAOSTAT/OAE ก่อนใส่ Intro; ตัวเลขย่อใน paper บางตัวต่ำกว่าจริง [ดู 03 §7].
7. **นิยาม vigor_score (H7)** — ชี้แจงให้ชัดว่า "vigor" หมายถึงต้นสมบูรณ์ หรือจำนวนยอด — เพราะมันเปลี่ยนว่าสูตรไหน "ชนะ" และตีความผลต่างกัน.
